# Day 5: CNN 优化（BatchNorm + Dropout）

## 学习内容

### 1. BatchNorm（批归一化）
```python
nn.BatchNorm2d(32)  # 对 32 个通道分别归一化
```

**作用**：
- 加速训练收敛
- 稳定梯度
- 允许更大的学习率

**位置**：通常放在卷积层之后、激活函数之前
```python
Conv → BatchNorm → ReLU → MaxPool
```

### 2. Dropout（随机丢弃）
```python
nn.Dropout(0.5)  # 训练时随机丢弃 50% 神经元
```

**作用**：
- 防止过拟合
- 强制网络学习更鲁棒的特征
- 只在训练时生效

**常用值**：
- 全连接层：0.5
- 卷积层：0.1-0.3

### 3. 优化器对比

| 优化器 | 学习率 | 特点 |
|--------|--------|------|
| Adam | 0.001 | 自适应学习率，自动调整 |
| SGD+momentum | 0.01 | 非自适应，需要手动设置 |

**为什么 SGD 学习率更大？**
- Adam 自适应调整，基础学习率可以小
- SGD 不调整，需要更大的学习率才能有效更新

### 4. 对比实验结果

| 模型 | 测试准确率 |
|------|-----------|
| 基础 CNN + Adam | 98.77% |
| 优化 CNN + Adam | 98.76% |
| 优化 CNN + SGD | 98.85% |

MNIST 太简单，差距不明显。复杂任务上 BatchNorm + Dropout 效果更显著。

### 5. Dropout 验证结果

| 模型 | 训练准确率 | 测试准确率 | 差距 |
|------|-----------|-----------|------|
| 无 Dropout | 99.28% | 98.93% | +0.35% |
| 有 Dropout | 98.45% | 99.21% | -0.75% |

**结论**：Dropout 训练时更难（准确率低），但泛化能力更强（测试准确率高）。

## 关键概念

| 概念 | 说明 |
|------|------|
| BatchNorm | 归一化，加速训练，稳定梯度 |
| Dropout | 随机丢弃，防止过拟合 |
| 过拟合 | 训练好测试差，模型死记硬背 |
| 泛化能力 | 模型对新数据的适应能力 |

## 验证问题

1. **BatchNorm 放在激活函数之前还是之后？**
   - 标准做法：之前（Conv → BN → ReLU）
   - 也有研究说之后也可以

2. **Dropout 0.5 和 0.9 的区别？**
   - 0.5：丢弃 50%，常用
   - 0.9：丢弃 90%，太激进，训练困难

3. **Adam 和 SGD 学习率不同的原因？**
   - Adam 自适应调整，基础学习率小
   - SGD 非自适应，需要更大的学习率
